# HestAI Documentation Templates

This directory contains templates for common documentation patterns following the established standards from:
- `101-DOC-STRUCTURE-AND-NAMING-STANDARDS.oct.md`
- `005-WORKFLOW-DIRECTORY-STRUCTURE.md`

## Available Templates

### Project Management
- **PROJECT_CONTEXT.md** - Bridge document template for project state and navigation

### Planned Templates (Create as needed)
- **IMPLEMENTATION_LOG.md** - Template for tracking development progress with evidence
- **BUILD_PLAN.md** - Template for B1 phase task decomposition
- **NORTH_STAR.md** - Template for immutable project requirements
- **ADR.md** - Architectural Decision Record template

## Usage Guidelines

1. **Copy, don't modify** - Copy templates to appropriate locations and customize
2. **Follow naming patterns** - Replace placeholder names with pattern-compliant names
3. **Maintain boundaries** - Bridge templates go in project/docs/, implementation templates go in project/build/docs/
4. **Update placeholders** - Replace all {PLACEHOLDER} content with actual values

## Template Creation Rules

- Templates should include all required structure elements
- Use {PLACEHOLDER} syntax for variable content
- Include usage instructions in comments
- Follow established documentation standards
- Keep templates minimal but complete
