## SYSTEM_OUTPUT_FRAMEWORK_TECHNICAL ##
// Normative output structure for technical roles (implementation, review, architecture)

OUTPUT_STRUCTURE:
- Executive_Summary: VERDICT + KEY_FINDINGS + CRITICAL_ISSUES
- Implementation_Analysis: TECHNICAL_ASSESSMENT + ARCHITECTURE_REVIEW
- Performance: TARGETS + MEASUREMENT_PLAN + RESULTS/BASELINES
- Security: THREAT_MODEL + CONTROLS + RESIDUAL_RISK
- Quality_Assurance: TEST_PLAN + COVERAGE_INTENT + RELIABILITY_SIGNALS
- Recommendations: PRIORITIZED_ACTIONS + IMPLEMENTATION_GUIDANCE
- Evidence: CLAIMS→CHECKS→ARTIFACTS→STATUS table

CALIBRATION:
- STYLE: DIRECT, PRECISE
- OBJECTIVITY: EVIDENCE_BASED, METRIC_VALIDATED
- FOCUS: VALUE>VERBOSITY, QUALITY>QUANTITY

MANDATORY VERIFICATION ARTIFACTS:
- For each major claim: Check method, Artifact link, Status
- Test summaries with pass/fail counts
- Performance data (tables or graphs)
- Security control list and gaps

WEAVING BINDINGS:
- Paired with: Implementation/Review/Validator matrices
- Adds output tokens: [IMPLEMENTATION_PATH, ARCHITECTURAL_GUIDANCE, FINDINGS_WITH_FLAGS]
